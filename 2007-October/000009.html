Hi
I assume that live weights (=whole weights) always are uploaded. This have been the rule in FishFrame and is mentioned in the comments in the exchange format description. (I admit that looking at the format description again (p. 12) it isn'n crystal clear for market sampling as live weight only is mentioned under Sea sampling. When COST isn't including the variance in connection with the conversion I consider the conversion from any condition to live weight as a national issue which should be handled before upload in order to make input data streamlined and not too complicated. Believe me, things gets rearly complex later on dealing with raising, etc. I suggest that only live (=fresh, =whole-)weights are uploaded and that it is clearly explained in the format description.

Hi,

No variance calculation of that sort in COST, it will complexify too much for a result probably marginal compared to the (im)precision we have on the catch figures.

The point is that we have to ascertain that information in SL match information in CL and that information in CL can be merged between countries.

As we cannot add gutted and whole fish together, we have two options
1 - add a field PRESENTATION as suggested in the WIKI. The problem is that we need a reference table to combine datasets with different presentations
2 - request that all information be transformed previously as whole fish by the national labs. The problem arises when gutted fish are sampled!!

My choice went to 1, I can live with 2 or better if someone has another solution, but we need to agree on a clear rule.
