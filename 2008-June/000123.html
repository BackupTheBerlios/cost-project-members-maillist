Hi,
Code attached. It's too simple to use for COST as it changes the proportion mature at age matrix rather than starting at the sample level. However it is a useful illustration of using the FLR package and outputs. FLR objects can be used in the existing assessment simulation routines which is the next step on from COST.

I think using David's model as the base for simulation will be time efficient. Is it possible to generate data that doesn't fit the model assumptions, e.g. use the mean and variance from an estimated effect but apply a different distribution, e.g.2 generate from a different growth model?
David Maxwell




We can certainly change growth models, or any other distributions. Of course if we knew the ones we are using were wrong we probably wouldn't have used them in the first place, but this is a good test of sensitivity to assumptions.
