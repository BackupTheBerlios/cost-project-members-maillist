 Dear Teunis,

many thanks for your effort and congratulations for this. 
I wish a bright future for this FF-COST Data Exchange Format Specification.

My only consideration today is that you provided us with a pure FF DEFS. In order to make it a real FF-COST DEFS, I will remove all the tables and appendices not discussed within COST from your files and I will extend to the whole world a certain number of fixed list in the appendix I. This means that :


*	I remove the 3 tables CC and the 8 tables from Acoustic Surveys stage 1 and stage 3. 
*	I keep the section on tables TS 
*	From the appendix,
	

	*	I remove the sections project as it is a free field. 
	*	I modify the reference for Area (too much restrictive to the ICES world), we must refer to the list available on the JRC website http://datacollection.jrc.ec.europa.eu/c/document_library/get_file?folderId=2502&name=DLFE-261.xls and on FAO website http://www.fao.org/fishery/area/search 
	*	I modify the reference for Fishing Gear to refer to ISSCFG ftp://ftp.fao.org/FI/DOCUMENT/cwp/handbook/annex/AnnexM1fishinggear.pdf
		
	*	I add the mention that the list of Rectangles only applies to ICES areas
		

	*	I add the international species reference list : ftp://ftp.fao.org/FI/stat/data/ASFIS_sp.zip 
	*	I remove all AS codes 
	*	There are other modalities for maturity scale, such as WKMAT (1-5) and IBTS (1-4), do we include them in the reference list? or do we leave this list opened. Same for Commercial Size category scale and Subsampling category.
		
	*	Do we keep the Fishing activity category National in the appendix empty?
		

The modifications suggested do not affect the structure agreed so far, only the presentation and some of the reference lists. You will find enclosed my proposal for the joint FF-COST DEFS. This is not meant to delay any further the final proposal. I imagine we can have a quick exchange on it to fix all possible mistakes and/or ommission.

Later, all discussions on the format occuring while developing the COST packages will have to first consider this DEFS as the reference. There must be rooms for improving this format and we'll have to get strong arguments and reach a full consensus to modify any part of it.

Best regards.

Joel

	Dear All
	New version "25 January 2008" attached. 
	
	No structural changes in format, only in descriptions and code reference lists:
	 1. I agree that AS and CC has not been discussed in the COST forum. I suppose that this is likely to happen in a future COST 2. So we will keep the FishFrame-only formats in a separate document until then.
	 2. I agree in all Joel's suggestions of links to FAO-codes lists and have added the link to the country code list as well. The links instead of the full lists makes appendix 1 more reader friendly (it can now be printed out without having to cut down a forest each time).
	 3. Some code lists like "FAC national", "Commercial Size Category Scale", "Project" are open for additions. This is noted in appendix 1.
	 4. Due to the removal of SL.ValidityCode some comments should be changed/deleted:
	      4.1 Comment 1 under SL: Text changed from "Mandatory if validitycode = 1" to "Mandatory if child HL records exist".
	      4.2 Comment 1 under HL:  Text removed: "HLs are not allowed is the parent SL record has ValidityCode = 4."
	 5. Does anybody know a good official link for the list of statistical rectangles? ICES only seem to have a shape-file. So the full list of rectangles is still in appendix 1.
	 6. Addition to the description of the sex-fields on HL and CA: "(Optional for "Unsexed")".
	Note that there will be two meetings for reviewing the "Fisheries Activity Category European" matrix soon, this will most likely result in modifications of lvl 6 and possibly 5. Hopefully this will fix these codes for a while. BTW: We should consider having both the minimum and maximum mesh size as part of the code - but lets save the discussion until after the meetings.
	I suggest that all structural changes from now on are logged in an appendix 3. It is our experience from many exchange format changes in FishFrame 1 to 4, that the users responsible for the data extraction and formatting scripts find this very useful.
	Best regards
	Teunis
