Hi David,

Our agreement on the "data status" should put the data in an appropriate 
form for analysis. The idea is that you should not worry with it.

The logic is:
1) read the data into R;
2) create the RAW objects, e.g. csData objects with csData methods;
3) run EDA and create VALIDATED objects, e.g. csDataVal with csDataVal 
methods;
4) post-stratify and create CONSOLIDATED objects, e.g. csDataCons with 
csDataCons methods.

The consolidated objects should be used directly for data analysis, so 
you should only need to extract the data.frames and implement your 
statistical methods.

I'm not sure where we are now but I guess the EDA package already sorted 
out bullet point 3). I'm not sure where they are regarding the 
post-stratification and creation of consolidated objects.
