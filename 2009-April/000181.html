Hi,

I also don't know what's the problem.

When you call library with the lib.loc argument, the function adds your 
lib.loc to a variable called .lib.loc (note the "." before lib.loc) 
adding that path to R's session search path. This procedure is not 
working for COSTcore, although it works for other packages.

.libPaths() does the same, adds the path to .lib.loc . So we're 
bypassing the problem by setting the search path with an alternative 
procedure.

I'll have to send a message to the R core team asking for guidance as I 
can't understand why this is happening.
