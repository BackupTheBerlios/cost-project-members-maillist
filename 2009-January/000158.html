Dear COST members,

you've been waiting for this for such a long time!
you'll find here (http://wwz.ifremer.fr/cost/documentation/examples) two 
comprehensive examples of use of the packages COSTcore and COSTeda. You 
must first upload the latest releases of these packages here : 
http://wwz.ifremer.fr/cost/software_packages

I feel you are disappointed because we're still not using COSTdbe or 
COSTmbe!! I can tell you that a lot has been done, so much that there is 
a lot to work on before there is a first release available for the users 
on a robust way. Mathieu will leave us for a while, and I'm confident 
we'll be able to get him back on track by mi-February and that we'll be 
able to release at least the first COSTdbe package by the end of 
February, with different examples on how to use the large amount of 
methods.  I hope it will be the same for COSTmbe. At least, we'll make 
sure Dorleta can compare different methods in the COSTsim package as 
soon as they are available.

During February, I would like to exchange on the theory of all the 
methods developed. A first way to do it would be for those who have 
developed methods to draw the algorithm used like countries have done 
for the PGCCDBS 2004 
(http://www.ices.dk/reports/acfm/pgccdbs/PGCCDBS04.pdf, section 6) 
together with updating a documentation with the formulas in the CVS doc 
folder. You can find the first on the COSTdbe CVS (/inst/doc folder).

Then March will be the last month for beta testing the methods and begin 
to write the final report.

I count on your commitment to the project during this last rush

Best regards

Joel
